### 🗓️ Week 1: 语言基础与核心骨架 (Survival & Skeleton)

**目标：** 掌握 Go 核心语法，搭建符合标准规范的项目骨架，打通 “React -> Clerk -> Go -> Supabase” 的链路。

#### Day 1-2: 语法急行军 (Go Syntax Sprint)

不要看厚书，只看怎么用。

- **学习资源：** [Go by Example 中文版](https://gobyexample-cn.github.io/)
- **必修知识点：**
  - **变量与基本类型：** `:=` 怎么用。
  - **Structs (结构体):** 这是 Go 的核心，用来映射你的 JSON 数据。
  - **Methods & Interfaces:** 理解怎么给结构体绑定方法。
  - **Goroutines & Channels:** **(重点)** 看懂 `go func()` 和 `chan`，这是后面写 WebSocket 的基础。
  - **JSON 处理:** `json.Marshal` 和 struct tags (`json:"name"`).
- **作业：** 写一个简单的 CLI 工具，读取一个 JSON 文件，修改里面的某个字段，然后保存回去。

#### Day 3: 标准项目搭建 (Project Layout)

不要把代码都塞在 `main.go` 里，一开始就按规范来。

- **参考标准：** Golang Standard Project Layout (简化版)。

- **执行动作：**

  1. 初始化模块：`go mod init xxxxx`

  2. 创建目录结构：

     ```
     /lowCode-go-backend
     ├── cmd
     │   └── server
     │       └── main.go       // 入口：只负责初始化数据库连接，启动 HTTP 服务
     ├── internal              // 私有代码，核心业务逻辑
     │   ├── api               // 接入层：处理 HTTP 请求，解析 JSON (Gin/Echo)
     │   │   ├── handler       // 具体的 Controller
     │   │   ├── middleware    // 认证(Auth)、日志(Logger)、CORS
     │   │   └── router.go     // 路由定义
     │   ├── core              // 业务逻辑层 (Service Layer)
     │   │   ├── user.go       // 用户逻辑 (登录注册)
     │   │   ├── project.go    // 项目逻辑 (保存 Schema)
     │   │   ├── builder.go    // 构建逻辑 (一键发布)
     │   │   └── editor_hub.go // 协同逻辑 (WebSocket Hub)
     │   ├── data              // 数据持久层 (Repository Layer)
     │   │   ├── db.go         // GORM 初始化
     │   │   └── schema.go     // 数据库模型定义 (User, Project)
     │   └── pkg               // 工具类 (JWT 生成、Hash 加密)
     ├── config                // 配置文件 (config.yaml)
     ├── go.mod
     └── Dockerfile
     ```

- **作业：** 跑通一个 "Hello World" 的 Gin Web Server。

#### Day 4-5: 接入 Clerk 与 Supabase (The Integration)

这是最容易卡住的地方，也是最能体现“全栈能力”的地方。

- **任务 A (数据库):**

  - 注册 Supabase，创建一个 PostgreSQL 数据库。

  - 在 Go 中引入 **GORM** (`gorm.io`)。

  - 定义模型：

    Go

    ```
    type Project struct {
        ID        uint   `gorm:"primaryKey"`
        UserID    string `gorm:"index"` // 对应 Clerk User ID
        SchemaJSON string // 你的低代码 JSON
    }
    ```

- **任务 B (鉴权):**

  - 前端 React 集成 Clerk，登录后获取 `Token`。
  - **Go 中间件编写：** 写一个 `AuthMiddleware`，解析 HTTP Header 里的 `Bearer Token`。验证通过后，把 `UserID` 塞进 Go 的 Context 里 (`c.Set("userID", uid)`).

- **作业：** 实现 `POST /api/project` (保存) 和 `GET /api/project` (读取)。只有登录用户能存取自己的数据。

#### Day 6-7: 缓冲与复盘

- **Code Review:** 检查你的错误处理。Go 里面到处都是 `if err != nil`，确保你没有忽略它们。
- **Docker 初探：** 写一个最简单的 `Dockerfile`，确保能在本地 `docker run` 跑起来。

### 🗓️ Week 2: 核心功能攻坚 (Real-time & Build)

**目标：** 实现最难的 WebSocket 协同编辑，以及“一键发布”构建服务。

#### Day 8-10: WebSocket 协同引擎 (The Hard Part)

这是你项目的“灵魂”。

- **学习资源：** Gorilla WebSocket Chat Example (GitHub 搜这个，直接读源码)。
- **核心实现 (Hub 模式):**
  1. **Client:** 每个连接是一个 `Client` 结构体，包含 `send` channel。
  2. **Hub:** 管理所有 Client。有一个 `broadcast` channel。
  3. **互斥锁 (Mutex):** 在 Hub 里加一个 `map[string]string` 记录谁锁定了哪个组件。
- **流程落地：**
  - React 发送 `{"type": "LOCK", "id": "btn1"}`。
  - Go 接收 -> 检查 Map -> 没被锁则锁定 -> 广播 `LOCKED` 消息。
  - React 发送 `{"type": "PATCH", "data": ...}`。
  - Go 接收 -> 验证锁 -> 广播给房间内其他人。
- **作业：** 能够开两个浏览器窗口，A 窗口选中组件，B 窗口看到该组件高亮且无法点击。

#### Day 11-12: 构建服务 (The Builder)

实现简易版的 Vercel。

- **原理：** Go 的 `os/exec` 包可以调用系统 shell 命令。
- **实现步骤：**
  1. 接收前端的 `Publish` 请求。
  2. **异步处理 (Goroutine):** 不要卡住 HTTP 请求。
  3. Go 代码逻辑：
     - 创建临时目录。
     - 把 JSON Schema 转换成 `App.tsx` 代码 (调用你前端已有的生成逻辑，或者简单点，直接用 Go 生成字符串)。
     - 执行 `npm install && npm run build`。
     - (进阶) 把 `dist` 文件夹上传到 S3 或简单的静态文件服务器。
- **作业：** 点击发布，服务器后台打印出构建日志，最后返回一个访问链接。

#### Day 13: 部署上线 (Deployment)

- **平台：** Railway (推荐，最省心) 或 Render。
- **配置：**
  - 把 `Dockerfile` 提交到 GitHub。
  - 在 Railway 上连接你的 Repo。
  - **重要：** 设置环境变量 (ENV Variables)，把 Supabase URL 和 Clerk 公钥填进去。不要硬编码在代码里！

#### Day 14: 终极测试与简历包装

- **测试：** 找个朋友，让他用他的电脑登录，和你一起协同编辑。
- **文档：** 完善 `README.md`，画出架构图（就像上面那个），列出 API 文档。
